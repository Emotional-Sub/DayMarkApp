# DayMark 答辩提纲

## 1. 项目介绍

DayMark 是一个每日打卡习惯追踪 APP，主要用于记录学习、运动、阅读等每日任务。用户登录后可以管理自己的打卡事件，并为事件添加图片记录。

## 2. 功能演示重点

- 登录：展示默认账号登录和记住密码。
- 新增事件：输入标题、内容、时间。
- 图片功能：演示拍照或从相册选择图片。
- 列表展示：展示事件标题、内容、时间、图片和打卡次数。
- 打卡：点击打卡按钮，次数增加。
- 编辑：修改事件信息后保存。
- 删除：弹出确认框后删除事件。

## 3. 技术说明

- 使用 Java 和 XML 开发原生 Android 应用。
- 使用 SQLite 保存用户和事件数据。
- 使用 SharedPreferences 实现记住密码。
- 使用 ListView 和 BaseAdapter 展示事件列表。
- 使用 Intent 调用系统相机和相册。

## 4. 可能被问到的问题

### 为什么使用 SQLite？

SQLite 是 Android 自带的轻量级数据库，适合保存本地用户数据和打卡事件，不需要网络或服务器，演示稳定。

### 记住密码如何实现？

使用 SharedPreferences 保存是否记住密码、用户名和密码。下次打开登录页时读取这些数据并自动填充。

### 图片如何保存？

拍照得到的图片会保存到应用私有目录，并把图片 Uri 保存到数据库。相册选择图片时保存系统返回的 Uri。

### 如何避免误删？

删除事件前使用 AlertDialog 弹出确认框，用户确认后才执行数据库删除。
