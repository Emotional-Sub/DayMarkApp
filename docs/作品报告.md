# 每日打卡习惯追踪 APP 作品报告

## 一、项目基本信息

- 项目名称：DayMark 每日打卡习惯追踪 APP
- 开发工具：Android Studio Panda 2 | 2025.3.2
- 开发语言：Java、XML
- 数据存储：SQLite、SharedPreferences
- 运行平台：Android 手机或模拟器

## 二、项目简介

DayMark 是一款用于记录日常习惯和学习生活目标的 Android 应用。用户登录后可以创建自己的打卡事件，为每个事件填写标题、内容和时间，并可以添加图片作为记录。应用支持对事件进行新增、编辑、删除和打卡操作，适合用于阅读、运动、学习、背单词等每日习惯管理场景。

## 三、功能设计

### 1. 登录与记住密码

应用启动后首先进入登录页面。用户可以使用默认账号登录，也可以注册新账号。登录页面提供“记住密码”选项，勾选后会把用户名、密码和勾选状态保存在 SharedPreferences 中，下次打开应用时自动填入。

### 2. 事件新增

用户进入主页面后，点击“新增打卡事件”按钮进入编辑页面。页面中可以填写事件标题、事件内容和打卡时间，也可以添加图片。点击保存后，事件会写入 SQLite 数据库。

### 3. 事件展示

主页面使用列表显示所有事件。每个事件展示标题、内容、时间、累计打卡次数和图片，满足题目中显示标题、内容、时间等信息的要求。

### 4. 事件编辑与删除

每个事件卡片中提供“编辑”和“删除”按钮。编辑功能会读取原事件信息并回填到编辑页，修改后保存到数据库。删除功能会弹出确认框，避免误删。

### 5. 打卡记录

用户点击事件卡片中的“打卡”按钮后，应用会更新该事件的累计打卡次数和最近打卡时间。主页面上方会显示事件总数和累计打卡次数。

### 6. 拍照与相册选择

在新增或编辑事件页面中，用户可以点击“拍照”调用系统相机，也可以点击“相册选择”从手机相册中选择图片。选择后的图片会立即显示在页面预览区域，并保存到事件记录中。

## 四、数据库设计

### users 表

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| id | INTEGER | 用户编号，主键自增 |
| username | TEXT | 用户名，唯一 |
| password | TEXT | 密码 |

### habits 表

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| id | INTEGER | 事件编号，主键自增 |
| title | TEXT | 事件标题 |
| content | TEXT | 事件内容 |
| time_text | TEXT | 事件时间 |
| image_uri | TEXT | 图片地址 |
| check_count | INTEGER | 累计打卡次数 |
| last_check_at | INTEGER | 最近打卡时间 |
| created_at | INTEGER | 创建时间 |

## 五、关键技术

1. Activity 页面跳转：使用 Intent 在登录页、主页面和编辑页之间传递数据。
2. SQLite 本地数据库：使用 SQLiteOpenHelper 创建和管理用户表、事件表。
3. SharedPreferences：保存记住密码状态、用户名和密码。
4. ListView 自定义适配器：展示打卡事件列表，并处理打卡、编辑、删除按钮。
5. 系统相机与相册：使用 MediaStore.ACTION_IMAGE_CAPTURE 调用相机，使用 ACTION_OPEN_DOCUMENT 选择图片。

## 六、界面与交互

应用界面采用浅色背景、绿色主色和卡片式布局。登录页突出应用名称和账号输入区域，主页面展示事件统计和事件列表，编辑页集中完成事件信息录入和图片选择。删除事件时提供确认弹窗，输入为空时在输入框显示错误提示，提升交互体验。

## 七、演示流程

1. 打开 APP，进入登录页。
2. 使用默认账号 demo / 123456 登录，或注册新账号后登录。
3. 勾选“记住密码”，退出后再次进入可看到账号密码已自动填入。
4. 在主页面查看已有打卡事件。
5. 点击“新增打卡事件”，填写标题、内容和时间。
6. 点击“拍照”调用相机，或点击“相册选择”选择图片。
7. 保存事件后回到主页面，查看新增事件信息。
8. 点击“打卡”按钮，观察累计次数变化。
9. 点击“编辑”修改事件内容。
10. 点击“删除”并确认删除事件。
